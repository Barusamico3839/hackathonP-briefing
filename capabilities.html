結論サマリ（30秒版）

構想の骨子は制度・原理の両面で「成立する」。勝負どころは2つ: ①ELMの表現力の弱さを前処理と少数サンプルでどこまで補えるか（＝Sim/実機での要実験）と②サーボ騒音のマイク混入。デバイスが無くても Solist-AI Sim でPC先行検証できるのが最大の保険。
  • 二脳構成（Solist-AI=常時ONの小脳 ／ Raspberry Pi=会話の大脳）はルール上OK。FAQ明記+SPI連携作品の前例あり。ただし審査軸「Solist-AI活用度」があるので小脳を主役に
  • お手教示・なで叩く分類・異常音検知は構造的に実現可能。教師あり/教師なし両対応+追加学習がサンプルコード付き公式サポート
  • 最大リスク: ELM（ランダム射影+線形読み出し）で子供の発話・音種識別が所望精度に届くか ／「やさしくなでる」は加速度センサ単体では検出困難（感圧/静電容量パッチ追加が必要）
  • MaAI（相槌・頷き）は MIT ライセンス版（_kyoto）が Pi で動く見込み。ただし実機レイテンシは要実測
✅ できる 🟡 たぶんできる（要検証） ❌ できない ❔ 不明

Solist-AI の正体

会話AIは動かない。動くのは3層ニューラルネットの学習・推論。センサ信号の分類・異常検知・パターン学習に特化した「常時ONの小さな脳」。

学習の仕組み = ELM（エクストリーム・ラーニング・マシン）

  • 3層NN。入力→隠れ層の重みはランダム固定、学習するのは出力層だけ → だから超軽量で「その場で学習」できる（誤差逆伝播を回す普通のNNと根本的に違う）
  • 教師なし学習（異常検知: 「正常」を現場学習→「いつもと違う」を検出）と教師あり学習（クラス識別・予測）の両対応。Simも両方用意されている
  • 現場学習+追加学習が公式サポートのフロー。「なでられた＝正解ラベル」に変換すれば「使うほどその子仕様に育つ」を教師あり追加学習で実現できる（素の強化学習は非対応）
  • 隠れ層が固定ゆえ表現力は限定的 → FFT/窓/正規化などの前処理で「線形分離しやすい特徴量」に落とす設計が前提

スペック（ML63Q2557）

  • CPU: Arm Cortex-M0+ / 最大48MHz（超省電力・超低スペック。LLMは論外）
  • AIアクセラレータ AxlCORE-ODL（ローム独自）: 3層NNの学習・推論+FFT等の前処理をハードウェアで実行。ソフト方式比 約1000倍高速、CPU負荷ほぼゼロ
  • メモリ: Flash 256KB ／ RAM 16KB ／ DataFlash 8KB — 大入力×多クラス×多モデル同時はRAMが律速
  • 消費電力: 48MHz動作で10mA、STOP時2µA → 低速+HALT待機、イベント時だけ全力の運用でぬいぐるみの電池と整合

マイコンの実力と制約

項目判定要点
教師あり学習（クラス識別）✅公式に4クラス物体識別の完全事例あり。「なで/叩く/抱っこ/放置」分類、「声パターン→動作クラス」に直接対応。出力はクラス番号で、動作生成は自前ファーム
教師なし学習（異常検知）🟡自己符号化型で「正常を学習→逸脱を異常度で検出」。泣き声/悲鳴に使えるが特定音種の識別精度は前処理次第で要実験
HW前処理（FFT/窓/正規化）✅AxlCORE-ODLがCPU負荷ほぼゼロで実行。ELMの弱さをこれで補うのが定石
追加学習API✅公式サンプル AnomalyDetectionDemo に初期学習・推論・逐次学習が実装済み。API詳細はMyROHM配下
入力次元の上限🟡実質目安 約512点。256点×3軸=768を扱う事例あり。正確な最大値は非公開
隠れ層/出力クラス数の上限❔公開資料に明記なし。RAM16KBとβサイズ依存と推定 → MyROHM資料 or ローム問い合わせ
センサ入力I/F✅12bit ADC×12ch（最大1Msps → 16kHz音声は余裕）、I2C/SPI/UART。マイクはアナログ前段（アンプ）を外付け
サーボ駆動✅PWMでホビーサーボ(50Hz)を直接駆動可 →「自分でお手」をMCU単独でも動かせる（評価ボードのピン引き出しは要確認）
Solist-AI Sim でPC先行検証✅デバイスなしでOK。CSV投入→学習/推論が1回約1秒。要MyROHM（無料）

音声系（声コマンド・泣き声/悲鳴検知）

項目判定要点
M0+単体のソフトMFCC❌FPU/SIMD無し。キーワード検出で秒オーダーかかり、常時ストリーミング不可
RAM16KBに音声バッファ❌1秒16kHz PCM=32KBで丸ごと入らない → フレーム単位のストリーミング特徴抽出が必須
AxlCORE HW-FFT + HW-ELM経由🟡これが唯一の現実解。ただし公式FFT前処理は振動/〜10kHz帯前提で、音声16kHz/256点FFTが想定内かは未確認。mel/DCTはHWに無く、スペクトログラムをそのままELMへ
小語彙・話者依存の声コマンド（お手/おかわり）🟡ELMは少クラスに強い（研究: 2クラス~100%、5クラス97.6%、10クラス75%）。2〜5語に絞る本構想はELMの得意領域。「その子の声を現場で追加学習」はELM逐次学習と本質一致。AxlCORE実機での再現は未実証→実験1
泣き声/悲鳴/呼びかけ検知🟡(A)教師なし=実装容易だが誤検知大 (B)教師あり2クラス=研究でbaby cry 92%等。M0+級での再現性は要実験→実験3
マイク入力経路✅ADCで16kHz収録は余裕。I2S/PDMは無いのでアナログMEMSマイク+プリアンプ→ADCが素直
会話側の音声認識（参考）✅会話はPi側の仕事。Pico2(M33/FPU)級なら標準の音声認識がリアルタイムで動く=解決済み領域
4

加速度センサ（なで/叩く/教示動作）

項目判定要点
ぬいぐるみ×IMU動作分類の前例✅PlushPal（MIT・子供が実演で教える）、慶應の研究、修論（RF 81.7%）など作例が複数実在
叩く/抱っこ/放置/腕を動かされた🟡加速度の得意領域（叩く=インパルス、抱っこ=重力軸変化、放置=DC）。全身ジェスチャ研究で13クラス77%。叩く強度の個人差は要検証
やさしく「なでる」❌加速度単体では困難。高精度な撫で識別の先行研究は全て感圧/静電容量/毛皮センサ使用。綿の詰め物が振動を減衰 → 静電容量(MPR121)や感圧(Velostat)パッチの追加が必要
その子への追加学習（パーソナライズ）🟡「特徴抽出は凍結し出力層だけ追加学習」に強い前例（3ショット83%）。ELM=出力層のみ更新なので原理的に適合。1ショットは55%と低く数ショット必要。破壊的忘却は要検証
3層NN(ELM)で足りる特徴量🟡HW-FFT or 時間特徴（RMS/ゼロ交差/軸間相関）。5クラス程度なら77-85%級は妥当。感圧+深層CNN級(94-98%)には届かない見込み
教示UI（お手を実演で覚えさせる）✅PlushPal等で「腕を動かして数回実演→登録」は子供向けに成立実績あり

MaAI（相槌・頷き）× Raspberry Pi

京大のOSS。相槌タイミング・頷き角度（30fps）・ターンテイキングを音声から予測する。「大きな脳」の中核候補。

項目判定要点
入手・コードのライセンス✅pip install maai。コードはMIT。0.2.11（2026-07更新）と活発
モデルのライセンス✅末尾 _kyoto 版がMIT（無印はCC-BY-NC-ND=非商用・改変不可）。最初から _kyoto を使う
頷き出力→サーボ✅pitch/yaw/roll角度の時系列@30fpsが出る → そのままサーボ角に写像。相槌タイミング p_bc も出る
Pi 4/5 でリアルタイム🟡公式実証はサーバCPUで14.6ms/frame。Piの実測値は世界に無い。frame_rate=10に落とせばPi5で間に合う見込み十分 → 実験4で実測
素のPiインストール🟡onnxruntime-gpuにARM64ホイールが無く素のpipは失敗見込み。torch先入れ+--no-deps等で回避（normalモデルはonnxruntime不要）
相槌「音声」の合成🟡MaAIはタイミング/種類の予測まで。相槌音声そのものは別途TTS/録音が必要
子供の発話での精度🟡学習は成人対話コーパス(76%)。子供・泣き声はドメイン外。韻律ベースで話者非依存性はあるが未検証

システム統合(2つの脳+サーボ+電源+通知)

項目判定要点
Pi↔MCU通信✅UART/SPI/I2C/GPIO、両者3.3Vで直結可。イベント通知は115200bps UARTで十分
評価ボードでの実配線🟡外部I/FはUSB-Cとセンサ用14ピン中心。付属ソフトはWindows専用でLinuxのPiから話せるかは要検証。基板125×66mmと大きめ。カスタムはブレークアウト版DT-BBML63Q2557が素直
サーボ制御✅Pi+PCA9685(I2C 16ch)が定番。サーボは別5V電源+共通GND必須。腕上げトルクはMG90S(2.2kg·cm)推奨
静音サーボ（マイク混入対策）🟡「音を聴く」製品なのでサーボ音のマイク混入が本質リスク。アナログ+プラギア小型+動作時以外トルクオフが現実解。実測必須 → 実験7
マイク2系統🟡独立2本が無難（Pi用=USB/I2S、MCU用=アナログ→ADC）。評価ボードのADC/フィルタが音声帯域に適合するかが最大の不確実点
バッテリー駆動🟡Pi4主体5〜6Wで10,000mAh≈5〜6時間。Pi5は5V/5A給電がバッテリーで困難 → Pi4かPi Zero 2 W推奨。MCUは40mWで無視できる
オフライン親通知（クラウドゼロ）✅自己ホストntfy + Android(F-Droid版) + ローカルWi-Fi(hostapd)でクラウド一切なしの即時push可。iOSはAPNs必須で例外

実験しないとわからないことリスト（優先順）

Sim系（1・3・5）はデバイス無しで今すぐ着手できる。

  1. Solist-AI Sim で「声→動作クラス」PoC デバイス不要ELM+FFTで2〜5語コマンドが実用精度(>90%)に届くか。PCマイクで各10-30発話→FFT特徴→教師あり版で交差検証（2-3日）
  2. 加速度4-5クラス分類オフライン検証なで/叩く/抱っこ/放置/腕動かしが分離できるか。MPU6050等で収集→RF/小型MLP→混同行列。なでは静電容量/感圧併用も比較（2-3日）
  3. 泣き声/悲鳴のanomaly検知PoC デバイス不要生活音を正常学習→泣き/悲鳴で異常度が上がり誤検知が許容内か。Sim異常検知版でROC評価（2-3日）
  4. MaAI を Pi にインストール&レイテンシ実測Pi4/5でnormal/_kyotoが起動するか、frame_rate=5/10/20でリアルタイムに間に合うか（1-2日）
  5. 隠れ層/クラス上限・追加学習API・メモリ収支 デバイス不要16KB RAMで「声3-5語+加速度4分類+異常音1」同時搭載できるか。MyROHM配下の資料精読+不足はローム問い合わせ（半日+回答待ち）
  6. デバイス見積・二脳構成の事務局確認キットが9/28に間に合うか、二脳構成が活用度で不利にならないか。EC見積 or データ・テクノ直電（半日+回答待ち）
  7. サーボ騒音のマイク混入テストSG90/MG90S/コアレスを実装しPiマイクで録音、トルクオフ効果も比較（1日・部品入手後）
  8. オフライン親通知E2Eネット遮断でPi自己ホストntfy→Androidへ数秒で届くか（半日-1日）
