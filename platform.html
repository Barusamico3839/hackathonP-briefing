Solist-AI プラットフォーム
開通ガイド

デバイスが無くても、今日からPCでAIの学習・推論を試せる。二人とも15〜30分で開通できる手順書。

0円で始められる
（登録もツールも全部無料）
1

開通手順（15〜30分／人）

ひかる・けいとそれぞれ自分のアカウントが必要。MTG中にその場でやってしまうのが早い。

  1. MyROHM に無料登録 rohm.co.jp/registration — 登録無料・メール認証あり。個人でも登録できる想定（会社名欄は所属 or 個人名でOK）
  2. ログインしてダウンロードページへ Solist-AI 製品ページのソフトウェア欄（一覧: 開発支援システム）
  3. Solist-AI Sim を2種類ともダウンロード 「教師あり学習版」（声→動作クラスのPoCに使う）と「教師なし学習版」（泣き声・悲鳴の異常検知に使う）。ついでに Solist-AI Scope とサンプルソフト・アプリケーションノートも落としておく
  4. Windows にインストールして起動確認 ⚠️ ツール群は Windows 専用。ひかるは WSL じゃなく Windows ホスト側に入れること。CSV を読ませて学習→推論が1周したら開通完了 🎉
なんで今日やるの？ デバイスの納期リスクと切り離して、アイデアの最大の技術リスク（ELMの表現力で精度が出るか）を先に検証できるから。Sim は1回約1秒で回るので、条件を変えてサクサク試せる。

3

開通したら最初に回す実験

デバイス不要でできる3本。どれも「作るか作らないか」の判断に直結する。

  1. 声→動作クラス PoC 教師あり版 PCマイクで「お手」「おかわり」等 2〜5語を各10〜30回録音 → FFT特徴量 → 交差検証。子供の声コマンドが実用精度(>90%)に届くかは本案の最大リスク。ここが白黒つくと一気に進む（工数2〜3日）
  2. 泣き声・悲鳴の異常検知 PoC 教師なし版 生活音を「正常」として学習 → 泣き声・悲鳴で異常度が上がるか、誤検知率は許容内か（工数2〜3日）
  3. スペック上限の確認 資料精読 MyROHM 配下のアプリケーションノート／APIマニュアルで、隠れ層・クラス数上限と追加学習APIの仕様を確認。RAM16KBに「声3〜5語＋加速度4分類＋異常音1」が同居できるかが知りたいこと（半日）
Claude Code との分担。 録音データの収集スクリプト・CSV整形・結果の集計は Claude が自動化できる（→ API・自動化調査）。人間がやるのは録音とSimのポチポチだけ。
