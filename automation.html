Solist-AI プラットフォームは
APIで操作できる？

答え: 公開APIは無い。でも入出力がファイルベースだから、Claude Code がまわりを全部自動化できる。

1

まず結論

Q. Solist-AI Sim を API や CLI から叩ける？

A. 叩けない。Sim の正体は MATLAB App Designer で作られ MATLAB Compiler でアプリ化された Windows GUI アプリ（ローム自身が2024年の講演資料で製造方法を解説している）。公開API・CLI・SDKは存在せず、操作はGUIのポチポチのみ。クラウドサービスでもないのでWeb APIも無い。

Q. じゃあ自動化は無理？

A. 入口と出口がファイルなので、実は大部分を自動化できる。Sim のワークフローは「CSV読み込み → 前処理設定（窓関数/FFT/正規化/エンベロープ）→ AI設定（隠れ層/活性化関数/誤差関数/学習・推論範囲）→ 実行（約1秒）→ 結果確認・閾値検討 → AIモデルファイル保存 → 実機に反映」。太字以外のGUI操作を挟む前後、つまりCSVの生成・整形と、出てきた結果・モデルファイルの解析は全部プログラムで自動化できる = Claude Code の担当領域。

Q. Claude Code は Solist-AI のことを分かって開発できる？

A. できる状態にした。アルゴリズム（ELM）の公開資料・データシート・講演資料を精読済みで、リポジトリの research/ に全部記録してある。さらに下の「Python再現実装」戦略で、Sim を介さずに Claude 自身がAIの挙動を再現・実験できるようにする。

2

本命戦略: Python で ELM を再現して実験を回す

AxlCORE-ODL のアルゴリズムは公式アプリケーションノートで公開されている。中身は「ランダム射影 + 出力層のみ学習」のELM — Pythonで数十行で書ける。

  1. Claude が Python で ELM 再現実装を書く入力→隠れ層はランダム固定、出力層は最小二乗で解く。FFT前処理も numpy で同等品を用意
  2. パラメータスイープを全自動で回す隠れ層数 × 入力次元 × 前処理の組合せを Claude Code が一晩で総当たり。交差検証・混同行列・ROC も自動生成。GUIのSimでは1条件ずつ手作業になるところ
  3. 有望な条件だけ公式 Sim で追試提出物で語る精度は公式ツール/実機の数字を使う（審査対策としても「実測データの説得力」に直結）
  4. 最終的に実機で確認して締めるSim → AIモデルファイル → 実機反映 → Scope で波形確認、という公式チェーンに乗せる
なぜこれが効くか。ELMは学習が軽い代わりに前処理と条件設計が精度のすべて。人間がGUIで数十条件を試すのと、Claudeが数千条件を回すのとでは勝負にならない。「実験1（声→動作クラス）」の成否判定が数日→数時間に縮む。

できる・できない早見表

やりたいこと可否方法・備考
Sim を API/CLI で実行❌公開APIなし。MATLAB Compiler製のGUIアプリ
入力データ(CSV)の生成・整形の自動化✅Claude Code が録音→特徴量→CSVまでスクリプト化
Sim の結果・モデルファイルの解析✅保存されるファイルをプログラムで読んで集計・可視化
AI実験そのものの自動化✅Python ELM再現実装でSimの外で回す（本命戦略）
GUI操作の自動化🟡Windows UIオートメーション（PowerShell+UIA等）で理論上可能だが壊れやすい。再現実装があるので優先度低
実機のプログラム制御🟡ファームはC APIライブラリ（初期学習/推論/逐次学習のサンプルあり・LEXIDE-Ωでビルド）。PCからはUSBシリアル経由 — 公式ホストアプリはWindows専用で、プロトコルを自前実装すればPi/Pythonから叩ける可能性（IOドライバのソース公開あり・要検証）
macOS / Linux でツールを動かす❌Sim/Scope/LEXIDE-Ω はWindows専用。WSLも不可、Windowsホスト側に入れる
MATLAB/Simulink 連携🟡ロームは社内でSimulinkの機械システムモデルを使って開発（講演資料）。一般提供のツールボックスは確認できず。MATLABライセンスが要るのでうちの規模では非現実的

人間とClaudeの分担（実験1を例に）

工程担当内容
録音👤 人間「お手」「おかわり」等を各10〜30回、PCマイクで発話（Claudeが録音スクリプトを用意）
特徴量化・CSV生成🤖 ClaudeFFT/窓/正規化を何パターンも自動生成
粗い探索🤖 ClaudePython ELM で数千条件スイープ→上位条件を報告
公式Simで追試👤 人間上位条件だけGUIで再現（数回のポチポチで済む）
集計・グラフ・記録🤖 Claude混同行列・精度表を作って research/ とこのサイトに反映
次のアクション: 二人のSim開通（開通ガイド）が済んだら、Claude が①録音スクリプト ②Python ELM再現実装 ③スイープハーネス を作る。MyROHM配下のAPIマニュアル（追加学習の仕様）が読めたら再現精度をさらに上げられるので、ダウンロードした資料は リポジトリ外の共有フォルダに置いて Claude に読ませてほしい（再配布NGの可能性があるためコミットはしない）。
